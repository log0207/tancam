1. SYSTEM ARCHITECTURE
----------------------
The system is built on a modular "Sense-Think-Act" architecture:

[Visual Input] -> [YOLOv8 Detection] -> [Simulation State] -> [PDWP Controller] -> [Traffic Signals]

A. Simulation Engine (`simulation.py`)
   - Function: Models physical vehicle movement, acceleration, and interaction.
   - Core Class: `Intersection` (Dimensions: 800x800).
   - Vehicle Dynamics:
     - `Vehicle` class tracks `x, y, speed, lane, wait_start_time`.
     - Gap maintenance: Vehicles stop if `dist_to_leader < VEHICLE_GAP` (15px).
     - Acceleration: +0.2 px/frame up to `MAX_SPEED` (3-5 px/frame).
   - Signal Interaction: Vehicles stop at `stop_line` if light is RED/YELLOW.

B. Detection Layer (`main.py`)
   - Model: Ultralytics YOLOv8n (Nano) for low-latency inference.
   - Logic: 
     - Processes 3 video feeds (Junctions 1, 2, 3) concurrently.
     - Maps bounding boxes to specific lanes using hardcoded ROI coordinates.
     - Calculates `Occupancy Score = count * weight` (Bus=3.0, Car=1.0, Bike=0.5).

C. Control Logic (`controller.py`)
   - The "Brain" of the operation.
   - State Machine: GREEN -> YELLOW_CLOSE (3s) -> CLEARANCE (2s) -> GREEN.
   - update() Frequency: 60Hz.

2. ALGORITHM LOGIC: PREDICTIVE DENSITY-WEIGHTED PRESSURE (PDWP)
--------------------------------------------------------------
The `PredictiveAdaptiveController` optimizes throughput using three dynamic mechanisms:

A. Pressure Metric (The Priority Score)
   Formula: `P = Density + a*WaitTime + b*Starvation + c*MaxWait`
   - `Density`: Bus-weighted vehicle count.
   - `WaitTime`: Cumulative waiting seconds of all cars in lane * 0.05.
   - `Starvation`: `Cycles_Skipped * 5.0` (Ensures fairness).
   - `MaxWait`: If any single car waits > 60s, adds significant penalty to force switch.

B. Dynamic Green Extension (Throughput Maximization)
   - Default Behavior: Signals have a `min_green` (20s).
   - Extension Trigger: At `time_left < 2.0s`, if `Flow_Score > 3.0` (Lane is busy):
     -> ADD 5.0 seconds to `green_duration`.
   - Limit: Extends up to `max_green_profile` (120s in peak hours).

C. Proactive Switching (Zero-Waste Logic)
   - "Saturation Check": If `Flow_Score < 0.3` (Lane empty) for > 1.0s:
     -> IMMEDIATELY terminate Green phase.
   - Does not wait for timer to expire. Saves 10-15s per cycle.

3. EDGE CASES & HANDLING
------------------------
The system is fastidiously designed to handle failure states and anomalies:

A. "The Parking Lot" (Gridlock)
   - Scenario: Max vehicles, 0 speed.
   - Logic: Pressure metric heavily weights `WaitTime`. Even if density is static, soaring wait times force priority rotation to clear the deadlock.

B. "The Ghost Car" (Sensor Glitch)
   - Scenario: Camera misses a stopped bike.
   - Handling: `Starvation` counter ensures that even undetected lanes (or low-detection lanes) get a turn after `STARVATION_LIMIT` (2 cycles).

C. "Flip-Flop" Prevention
   - Scenario: Two lanes have equal, oscillating pressure.
   - Logic: Minimum Green Time (20s) acts as a hysteresis damper. A phase cannot be preempted until 8s have passed (except for Emergency).

D. Emergency Preemption
   - Scenario: Ambulance detected.
   - Action: 
     1. Identify `emg_lane`.
     2. If `current_green != emg_lane`: force switch to YELLOW immediately (after 8s damper).
     3. Skip other phases.
